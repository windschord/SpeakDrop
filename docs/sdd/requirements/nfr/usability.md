# ユーザビリティ要件

> 非機能要件カテゴリ: ユーザビリティ

## 概要

SpeakDropはmacOSのメニューバー常駐アプリとして、ユーザーの作業を妨げない直感的なUIを提供する。状態変化の即時フィードバックと、対応OSバージョンの明確な定義により、快適な使用体験を実現する。

## 要件一覧

### NFR-007: メニューバーアイコンの応答性

**EARS記法**: システムはメニューバーアイコンの状態変化（待機/録音中/処理中）を、状態遷移後200ms以内に反映しなければならない

**測定基準**:
- 指標: 状態遷移イベント発生からアイコン更新完了までの経過時間
- 目標値: 200ms以内
- 測定方法: 状態遷移ログのタイムスタンプ差分で計測

**対象状態遷移**:
- 待機中 → 録音中（プッシュトークキー押下時）
- 録音中 → 処理中（プッシュトークキー離放時）
- 処理中 → 待機中（テキスト挿入完了時）

**関連する機能要件**: REQ-003, REQ-004

**優先度**: 高

**検証方法**:
- [ ] 各状態遷移でアイコン更新のタイムスタンプを計測し200ms以内を確認
- [ ] 連続して複数回プッシュトークを行い、状態表示のずれがないことを確認

---

### NFR-008: 対応OSバージョン

**EARS記法**: システムはmacOS 13 (Ventura) 以降のApple Silicon Macで動作しなければならない

**測定基準**:
- 指標: 対象プラットフォームでの起動・動作確認
- 目標値: macOS 13以降、Apple Silicon（M1/M2/M3/M4シリーズ）で全機能が動作
- 測定方法: 対象環境での統合テスト実施

**対象プラットフォーム**:
- macOS 13 (Ventura) 以降
- Apple Silicon（M1以上）
- Intel Macは対象外

**関連する機能要件**: 全REQ

**優先度**: 高

**検証方法**:
- [ ] macOS 13 (Ventura) で全ユーザーストーリーの受入基準が満たされることを確認
- [ ] macOS 14 (Sonoma) で全ユーザーストーリーの受入基準が満たされることを確認
- [ ] Apple Silicon Mac（M1以上）での動作確認
